# rtl_433 — ISM Band Device Decoder

## What Is rtl_433?

rtl_433 is a command-line tool that receives and decodes signals from a wide variety of wireless devices operating in the ISM (Industrial, Scientific, and Medical) bands, primarily at **433.92 MHz** (Europe/worldwide) and **915 MHz** (US). It uses an RTL-SDR dongle (or other supported SDR hardware) as the receiver.

These ISM-band devices are everywhere: weather stations, wireless thermometers, tire pressure monitoring systems (TPMS), door/window sensors, soil moisture sensors, rain gauges, remote controls, and hundreds of other devices.

rtl_433 decodes the signals and outputs structured data (JSON, CSV, etc.) that can be logged, analyzed, or fed into home automation systems.

**Repository**: https://github.com/merbanan/rtl_433

---

## What It Can Decode

rtl_433 supports **200+ device protocols**. Some common categories:

### Weather Stations

- Acurite (various models: 5-in-1, Atlas, Tower sensors)
- Ambient Weather
- Fine Offset / Ecowitt
- La Crosse
- Oregon Scientific
- TFA Dostmann
- Bresser
- Davis Vantage

### Temperature / Humidity Sensors

- Acurite 06002M, 592TXR
- Fine Offset WH2, WH25
- Ambient Weather F007TH
- La Crosse TX141
- Oregon Scientific THN132N
- Numerous generic 433 MHz sensors

### Tire Pressure Monitoring (TPMS)

- Schrader
- Various automotive TPMS sensors (Ford, GM, Toyota, etc.)

### Door / Window Sensors

- Honeywell 5800 series
- DSC
- Generic 433 MHz security sensors
- Interlogix

### Soil / Garden Sensors

- Soil moisture sensors
- Rain gauges
- Pool temperature sensors

### Other Devices

- Smoke detectors
- Remote controls (garage doors, power outlets)
- Utility meters (some AMR/AMI)
- Lightning detectors
- Various industrial sensors

Run `rtl_433 -R help` for a complete list of supported protocols.

## Basic Usage

### Default Operation (433.92 MHz)

```bash
rtl_433
```

This starts listening on the default frequency (433.92 MHz) and prints decoded device data to the console. Leave it running and watch for your devices to transmit.

### Specify Frequency

```bash
# Listen on 433.92 MHz (default, European ISM)
rtl_433 -f 433920000

# Listen on 315 MHz (US TPMS, some remotes)
rtl_433 -f 315000000

# Listen on 868 MHz (European, some weather stations)
rtl_433 -f 868000000

# Listen on 915 MHz (US ISM)
rtl_433 -f 915000000

# Hop between multiple frequencies
rtl_433 -f 433920000 -f 915000000 -H 30
# -H 30 = hop every 30 seconds
```

### Specify Device Gain

```bash
# Set gain to 40 dB
rtl_433 -g 40

# Auto gain
rtl_433 -g 0
```

### Limit to Specific Protocols

```bash
# Only decode protocol 40 (Acurite 5-in-1 weather station)
rtl_433 -R 40

# Decode protocols 40 and 41
rtl_433 -R 40 -R 41

# Disable a protocol (enable all except)
rtl_433 -G -R -40
```

### Verbose Output

```bash
# Show signal details
rtl_433 -v

# Very verbose (debug level)
rtl_433 -vv
```

## Output Formats

### JSON Output

```bash
# Output as JSON (one object per line)
rtl_433 -F json

# Output JSON to a file
rtl_433 -F json:sensors.json
```

Example JSON output:
```json
{
  "time": "2024-03-15 14:32:01",
  "model": "Acurite-Tower",
  "id": 12345,
  "channel": "A",
  "battery_ok": 1,
  "temperature_C": 22.3,
  "humidity": 45
}
```

### CSV Output

```bash
rtl_433 -F csv:sensors.csv
```

### KV (Key-Value) Output (Default)

```bash
rtl_433 -F kv
```

Output looks like:
```
time      : 2024-03-15 14:32:01
model     : Acurite-Tower
id        : 12345
channel   : A
battery_ok: 1
temperature_C: 22.3
humidity  : 45
```

### Combine Multiple Outputs

```bash
# Print to console AND log JSON to file
rtl_433 -F kv -F json:sensors.json
```

## MQTT Integration

rtl_433 can publish decoded data directly to an MQTT broker, making it easy to integrate with home automation systems (Home Assistant, Node-RED, OpenHAB, etc.).

### Basic MQTT Output

```bash
rtl_433 -F "mqtt://localhost:1883,events=rtl_433/events"
```

This publishes all decoded events to the MQTT topic `rtl_433/events`.

### Structured MQTT Topics

```bash
rtl_433 -F "mqtt://localhost:1883,devices=rtl_433/devices[/model][/id]"
```

This creates per-device topics like `rtl_433/devices/Acurite-Tower/12345`.

### With Authentication

```bash
rtl_433 -F "mqtt://user:password@broker.example.com:1883,events=rtl_433/events"
```

### Home Assistant Integration

With MQTT discovery enabled, Home Assistant can automatically discover rtl_433 sensors:

```bash
rtl_433 -F "mqtt://localhost:1883,events=rtl_433/events,devices=rtl_433/devices[/model][/id]" \
        -M time:utc -M protocol -M level
```

Then use the rtl_433 MQTT Auto Discovery script or integration to create HA entities.

Community project for HA integration: https://github.com/pbkhrv/rtl_433-hass-addons

## Configuration File

For persistent configuration, create a config file:

```bash
# Default location
~/.config/rtl_433/rtl_433.conf

# Or specify explicitly
rtl_433 -c /path/to/config.conf
```

Example config file:

```ini
# rtl_433 configuration

# Frequency to listen on
frequency 433920000

# Also hop to 915 MHz
frequency 915000000
hop_interval 30

# Gain
gain 40

# Output formats
output json:///var/log/rtl_433/sensors.json
output mqtt://localhost:1883,events=rtl_433/events

# Only enable specific protocols (uncomment as needed)
# protocol 40    # Acurite 5n1
# protocol 41    # Acurite Tower

# Report metadata
report_meta time:utc
report_meta protocol
report_meta level
```

## Running as a Service

### systemd Service (Linux)

Create `/etc/systemd/system/rtl_433.service`:

```ini
[Unit]
Description=rtl_433 ISM Band Decoder
After=network.target

[Service]
Type=simple
ExecStart=/usr/local/bin/rtl_433 -c /etc/rtl_433.conf
Restart=on-failure
RestartSec=10
User=nobody

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable rtl_433
sudo systemctl start rtl_433
sudo systemctl status rtl_433
```

## Practical Examples

### Find All Active Devices Nearby

```bash
# Run rtl_433 with all protocols enabled, verbose output
rtl_433 -g 40
```

Leave it running for 10-30 minutes. Most wireless sensors transmit periodically (every 30-60 seconds for weather stations). You will see a variety of neighbor's weather stations, car TPMS, and other devices.

### Monitor a Specific Weather Station

```bash
# Find your weather station's protocol number and ID first (run default mode)
# Then filter to just that device
rtl_433 -R 40 -F json | grep '"id":12345'
```

### Log All Data to File with Timestamps

```bash
rtl_433 -g 40 -M time:utc -F json:/home/pi/sensor_log.json
```

### Real-Time Temperature Dashboard

Pipe JSON output to a script that extracts temperature:

```bash
rtl_433 -F json | python3 -c "
import sys, json
for line in sys.stdin:
    try:
        data = json.loads(line)
        if 'temperature_C' in data:
            print(f\"{data.get('model','?')}: {data['temperature_C']}C / {data['temperature_C']*9/5+32:.1f}F\")
    except:
        pass
"
```

## Troubleshooting

### No Devices Detected

- Verify your RTL-SDR is working: `rtl_test -t`
- Increase gain: `rtl_433 -g 49`
- Make sure you are on the correct frequency for your region (433.92 for EU, 915 for US).
- Some devices transmit infrequently — wait longer.
- Check antenna connection. The stock RTL-SDR antenna works at 433 MHz but a quarter-wave wire antenna (17.3 cm) works better.

### Too Much Data / Neighbor's Devices

- Filter by protocol: `-R PROTOCOL_NUMBER`
- Filter by device ID in your processing pipeline (JSON output + grep/jq).

### "No supported devices found"

- RTL-SDR drivers not installed or DVB-T modules not blacklisted.
- See rtl-sdr-setup.md for driver installation.

### Known Limitations

- rtl_433 cannot decode encrypted or rolling-code protocols (most modern garage door openers, car key fobs).
- Range depends on your antenna and the transmitter. Typical: 30-300 feet for small sensors, potentially further for weather stations with outdoor transmitters.
- Some devices use 868/915 MHz — set the correct frequency.
