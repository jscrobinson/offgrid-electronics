# Character LCD Guide (1602 / 2004)

## Overview

Character LCDs based on the Hitachi HD44780 controller (or compatible clones like SPLC780D, KS0066U) are the workhorses of embedded displays. They are cheap, widely available, easy to use, and readable in direct sunlight. The two most common sizes are 16x2 (16 characters, 2 rows) and 20x4 (20 characters, 4 rows).

## Specifications

| Parameter | 1602 | 2004 |
|---|---|---|
| Characters | 16 columns x 2 rows | 20 columns x 4 rows |
| Controller | HD44780 or compatible | HD44780 or compatible |
| Character size | 5x8 pixels (5x10 mode rarely used) |  5x8 pixels |
| Interface | Parallel 4-bit or 8-bit, or I2C via backpack | Same |
| Operating voltage | 5V (3.3V versions exist but are less common) | 5V |
| Backlight | LED, typically white or green/blue | LED |
| Backlight current | ~20-60 mA | ~40-80 mA |
| Operating current (no BL) | ~1-2 mA | ~1-2 mA |

## Pin Description (Parallel Interface)

The standard HD44780 LCD module has 16 pins:

| Pin | Name | Function |
|---|---|---|
| 1 | VSS | Ground |
| 2 | VDD | +5V power |
| 3 | V0 (VEE) | Contrast adjustment (connect to potentiometer wiper) |
| 4 | RS | Register Select: 0=command, 1=data |
| 5 | R/W | Read/Write: 0=write, 1=read (usually tied to GND for write-only) |
| 6 | E | Enable: data latched on falling edge |
| 7 | D0 | Data bit 0 (not used in 4-bit mode) |
| 8 | D1 | Data bit 1 (not used in 4-bit mode) |
| 9 | D2 | Data bit 2 (not used in 4-bit mode) |
| 10 | D3 | Data bit 3 (not used in 4-bit mode) |
| 11 | D4 | Data bit 4 |
| 12 | D5 | Data bit 5 |
| 13 | D6 | Data bit 6 |
| 14 | D7 | Data bit 7 |
| 15 | A (LED+) | Backlight anode (+5V or via resistor/transistor) |
| 16 | K (LED-) | Backlight cathode (GND) |

## 4-Bit vs 8-Bit Mode

### 8-Bit Mode
Uses all data pins D0-D7 (8 data pins + RS + E + R/W = 11 pins, or 10 if R/W is grounded). Sends a full byte in one clock cycle.

### 4-Bit Mode (Recommended)
Uses only D4-D7 (4 data pins + RS + E = 6 pins, with R/W tied to GND). Each byte is sent as two nibbles (high nibble first, then low nibble). Slightly slower but saves 4 GPIO pins. This is the standard approach.

### 4-Bit Wiring to Arduino Uno

```
LCD Pin    Arduino Pin    Notes
-------    -----------    -----
VSS (1)    GND
VDD (2)    5V
V0  (3)    Potentiometer wiper (10K pot between 5V and GND)
RS  (4)    D12            (any digital pin)
R/W (5)    GND            (write-only mode)
E   (6)    D11            (any digital pin)
D4  (11)   D5             (any digital pin)
D5  (12)   D4             (any digital pin)
D6  (13)   D3             (any digital pin)
D7  (14)   D2             (any digital pin)
A   (15)   5V             (or through 220 ohm resistor, or via transistor for control)
K   (16)   GND
```

## I2C Backpack Module (Recommended)

The I2C LCD backpack uses a **PCF8574** (or PCF8574A) I/O expander chip to reduce the interface to just 2 wires (SDA + SCL), plus power. This is by far the easiest way to use these LCDs.

### I2C Addresses

| Chip | Default Address | Address Range |
|---|---|---|
| PCF8574 | 0x27 | 0x20 - 0x27 (set by A0-A2 solder pads) |
| PCF8574A | 0x3F | 0x38 - 0x3F (set by A0-A2 solder pads) |

If you are unsure of the address, run an I2C scanner.

### I2C Wiring

| I2C Backpack Pin | Connection |
|---|---|
| VCC | 5V |
| GND | GND |
| SDA | SDA (Arduino Uno: A4, ESP32: GPIO21) |
| SCL | SCL (Arduino Uno: A5, ESP32: GPIO22) |

The backpack module has a blue potentiometer on it for contrast adjustment. The backlight is controlled by a jumper on the board (remove to disable) or via software.

## Arduino — Parallel Mode

### LiquidCrystal Library (built-in)

```cpp
#include <LiquidCrystal.h>

// LiquidCrystal(RS, E, D4, D5, D6, D7)
LiquidCrystal lcd(12, 11, 5, 4, 3, 2);

void setup() {
  lcd.begin(16, 2);  // 16 columns, 2 rows (use 20,4 for 2004)
  lcd.print("Hello, World!");
  lcd.setCursor(0, 1);  // column 0, row 1
  lcd.print("Line 2 here");
}

void loop() {
  // Update display
  lcd.setCursor(10, 1);
  lcd.print(millis() / 1000);
}
```

## Arduino — I2C Mode

### LiquidCrystal_I2C Library

Install via Library Manager: search for "LiquidCrystal I2C" by Frank de Brabander, or "LiquidCrystal_I2C" by Marco Schwartz.

```cpp
#include <Wire.h>

// LiquidCrystal_I2C(address, columns, rows)
LiquidCrystal_I2C lcd(0x27, 16, 2);  // change to 0x3F if needed

  lcd.init();        // initialize the LCD
  lcd.backlight();   // turn on backlight

  lcd.setCursor(0, 0);
  lcd.print("Hello World!");
  lcd.setCursor(0, 1);
  lcd.print("I2C LCD Works!");
}

void loop() {
  // nothing
}
```

### Common I2C LCD Functions

```cpp
lcd.init();              // Initialize LCD
lcd.backlight();         // Backlight ON
lcd.noBacklight();       // Backlight OFF
lcd.setCursor(col, row); // Set cursor position (0-indexed)
lcd.print("text");       // Print string
lcd.print(variable);     // Print number/variable
lcd.clear();             // Clear display and home cursor
lcd.home();              // Move cursor to (0,0) without clearing
lcd.scrollDisplayLeft(); // Scroll entire display left
lcd.scrollDisplayRight();
lcd.blink();             // Blinking cursor
lcd.noBlink();
lcd.cursor();            // Underline cursor
lcd.noCursor();
lcd.noDisplay();         // Turn off display (data retained)
lcd.display();           // Turn on display
```

## Contrast Adjustment

### With Potentiometer (Standard Method)

Connect a 10K potentiometer:
- One outer pin to 5V
- Other outer pin to GND
- Wiper (center) pin to LCD V0 (pin 3)

Turn the pot until text is clearly visible. If you see solid blocks on row 1, contrast is too high. If you see nothing, contrast may be too low.

### With PWM (No Potentiometer)

```cpp
#define CONTRAST_PIN 9  // PWM pin

void setup() {
  pinMode(CONTRAST_PIN, OUTPUT);
  analogWrite(CONTRAST_PIN, 50);  // 0-255, typically 50-100 works well
  // Connect this pin to V0 on the LCD
}
```

### On I2C Backpack

Turn the small blue potentiometer on the backpack board itself using a small screwdriver.

## Custom Characters

The HD44780 can store up to 8 custom characters (indices 0-7) in CGRAM. Each character is 5 pixels wide by 8 pixels tall.

```cpp
#include <LiquidCrystal_I2C.h>
LiquidCrystal_I2C lcd(0x27, 16, 2);

// Define custom characters (5 bytes per char, each byte = 1 row, lower 5 bits used)
byte heart[8] = {
  0b00000,
  0b01010,
  0b11111,
  0b11111,
  0b11111,
  0b01110,
  0b00100,
  0b00000
};

byte thermometer[8] = {
  0b00100,
  0b01010,
  0b01010,
  0b01010,
  0b01010,
  0b11111,
  0b11111,
  0b01110
};

byte battery[8] = {
  0b01110,
  0b11111,
  0b10001,
  0b10001,
  0b10001,
  0b11111,
  0b11111,
  0b11111
};

void setup() {
  lcd.init();
  lcd.backlight();

  lcd.createChar(0, heart);
  lcd.createChar(1, thermometer);
  lcd.createChar(2, battery);

  lcd.setCursor(0, 0);
  lcd.write(0);  // display heart
  lcd.print(" Temp: 24");
  lcd.write(223); // degree symbol (built-in)
  lcd.print("C");

  lcd.setCursor(0, 1);
  lcd.write(1);  // thermometer icon
  lcd.print(" ");
  lcd.write(2);  // battery icon
  lcd.print(" 85%");
}

void loop() {}
```

**Tip:** Use an online custom character generator like https://maxpromer.github.io/LCD-Character-Creator/ (save this page offline) to design characters visually.

**Note:** Custom char index 0 can be tricky. With some libraries, `lcd.write(0)` may not work. Use `lcd.write((uint8_t)0)` or `lcd.write(byte(0))` instead.

## ESP32 Notes

```cpp
#include <Wire.h>
#include <LiquidCrystal_I2C.h>

LiquidCrystal_I2C lcd(0x27, 20, 4);

void setup() {
  Wire.begin(21, 22);  // SDA, SCL — specify for ESP32
  lcd.init();
  lcd.backlight();
  lcd.print("ESP32 LCD!");
}

void loop() {}
```

**Important:** The standard HD44780 LCD is a 5V device. The ESP32 is 3.3V. The I2C backpack is typically powered at 5V but the I2C lines may need level shifting. In practice, many PCF8574 backpacks work fine with 3.3V I2C signals from the ESP32 because the PCF8574's I2C threshold is compatible. If you have issues, use a bidirectional level shifter on SDA/SCL.

## MicroPython

### I2C LCD with MicroPython

Use the `lcd_api` and `i2c_lcd` libraries. These are not built-in; copy them to your board.

```python
from machine import Pin, I2C
from lcd_api import LcdApi
from i2c_lcd import I2cLcd

I2C_ADDR = 0x27
I2C_NUM_ROWS = 2
I2C_NUM_COLS = 16

i2c = I2C(0, sda=Pin(21), scl=Pin(22), freq=400000)

lcd = I2cLcd(i2c, I2C_ADDR, I2C_NUM_ROWS, I2C_NUM_COLS)

lcd.clear()
lcd.putstr("Hello from\nMicroPython!")

# Custom character
heart = bytearray([0x00, 0x0A, 0x1F, 0x1F, 0x1F, 0x0E, 0x04, 0x00])
lcd.custom_char(0, heart)
lcd.move_to(0, 1)
lcd.putchar(chr(0))

# Backlight control
lcd.backlight_on()
lcd.backlight_off()
```

## 20x4 LCD Memory Map

The HD44780 has an internal 80-byte display RAM. For a 20x4 display, the row starting addresses are NOT sequential:

| Row | DDRAM Address Start | Address Range |
|---|---|---|
| Row 0 | 0x00 | 0x00 - 0x13 |
| Row 1 | 0x40 | 0x40 - 0x53 |
| Row 2 | 0x14 | 0x14 - 0x27 |
| Row 3 | 0x54 | 0x54 - 0x67 |

This means rows 0 and 2 are contiguous in memory, as are rows 1 and 3. If you print a string that overflows row 0, it wraps to row 2 (not row 1). Libraries handle this for you with `setCursor()`.

For a 16x2 display: Row 0 starts at 0x00, Row 1 at 0x40.

## Backlight Control

### Hardware Control

To control the backlight with a GPIO (for dimming or on/off):

1. Connect the backlight anode (pin 15/A) through a 220-ohm resistor to the collector of an NPN transistor (like 2N2222 or BC547).
2. Connect emitter to GND.
3. Connect base through a 1K resistor to a digital/PWM GPIO pin.

```cpp
#define BL_PIN 6  // PWM capable pin
analogWrite(BL_PIN, 128);  // 50% brightness
```

### Software Control (I2C Backpack)

The I2C backpack controls the backlight via a transistor connected to one of the PCF8574 outputs:

```cpp
lcd.backlight();    // ON
lcd.noBacklight();  // OFF
```

There is no PWM dimming via the I2C backpack — it is either on or off.

## Common Issues and Troubleshooting

### Blank Display (No Text Visible)

1. **Contrast not adjusted.** This is the #1 issue. Turn the contrast potentiometer (or the one on the I2C backpack) until you can see faint rectangles on row 1. Then fine-tune.
2. **Wrong I2C address.** Run an I2C scanner. The address might be 0x27 or 0x3F (or something else if address pins are modified).
3. **5V power required.** The HD44780 needs 5V. It will not work reliably on 3.3V (characters will be very faint or invisible). Make sure VDD is connected to 5V.
4. **R/W pin not grounded.** In parallel mode, pin 5 (R/W) must be connected to GND if you are only writing to the display.
5. **Initialization issue.** The LCD must be initialized correctly (4-bit mode requires specific sequence). Using a library handles this. If using bare code, follow the HD44780 datasheet initialization flow exactly.

### Row 1 Shows Black Blocks, Row 2 Blank

This is normal when the LCD is powered on but not initialized. It means the LCD is working and contrast is set correctly. Your code just has not sent the initialization commands yet.

### Garbled Characters

1. **Loose connections.** Check all wires, especially D4-D7 in parallel mode. One bad connection causes garbled output.
2. **Wrong pin assignments in code.** Double-check that the pin numbers in your constructor match your physical wiring.
3. **Timing issues.** Some clones need longer delays. Try adding a 100ms delay after `lcd.begin()` or `lcd.init()`.
4. **Noise on long wires.** Keep wires short. Add 100nF capacitor across VCC/GND near the LCD.

### I2C Backpack: Address Not Found

1. Check that VCC is 5V (not 3.3V).
2. Verify SDA and SCL are not swapped.
3. Check for pull-up resistors (most backpacks have them built-in).
4. Try a different I2C speed: `Wire.setClock(100000);`

### Only 16 Characters Show on 20x4

You initialized the display as 16x2 in code. Change `lcd.begin(16, 2)` to `lcd.begin(20, 4)`.

## Tips

- **Scrolling text:** For strings longer than the display width, implement a scrolling marquee by shifting a substring window across your text and updating the display periodically.
- **Avoid `lcd.clear()` in loops.** It causes visible flicker. Instead, overwrite old text with spaces or use fixed-width formatting: `sprintf(buf, "%-16s", text);`
- **Degree symbol:** The built-in character at position 223 (0xDF) is the degree symbol: `lcd.write(223);`
- **Update rate:** Do not update the display more than about 5-10 times per second. The HD44780 is slow, and rapid writes cause flickering.
- **Power saving:** Turn off the backlight when not needed. The LCD itself draws very little current (1-2 mA). The backlight draws 20-80 mA depending on size.
